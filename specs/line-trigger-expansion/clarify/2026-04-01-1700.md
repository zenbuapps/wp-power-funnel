# Clarify Session 2026-04-01 17:00

## Idea

為 Power Funnel 工作流引擎新增更多 LINE 相關的觸發時機點（TriggerPoint）。

新增的 LINE TriggerPoints（按優先級）：

P0：LINE_POSTBACK_RECEIVED (pf/trigger/line_postback_received) — 收到 LINE Postback 事件
P1：LINE_JOIN (pf/trigger/line_join) — Bot 被加入群組/聊天室
P1：LINE_LEAVE (pf/trigger/line_leave) — Bot 被移出群組/聊天室
P1：LINE_MEMBER_JOINED (pf/trigger/line_member_joined) — 新成員加入 Bot 所在群組
P1：LINE_MEMBER_LEFT (pf/trigger/line_member_left) — 成員離開 Bot 所在群組
P2：LINE_BEACON (pf/trigger/line_beacon) — 用戶進入 Beacon 區域

技術要點：
- WebhookService 已 dispatch power_funnel/line/webhook/{type} hook
- 遵循 Serializable Context Callable 模式
- Postback 會與既有報名流程雙重觸發（正確行為）
- 群組事件 source 無 userId
- memberJoined 多人時每人獨立觸發
- RecursionGuard MAX_DEPTH=3 防護 Postback 迴圈

## Q&A

- Q: LINE_POSTBACK_RECEIVED 是否需要在 trigger_point meta params 中支援 postback_action 過濾？ → A: B — 在 trigger_point meta params 中支援 postback_action 過濾（類似 ACTIVITY_BEFORE_START 的 before_minutes 參數）
- Q: P1 群組事件（JOIN/LEAVE/MEMBER_JOINED/MEMBER_LEFT）是否有具體業務場景？ → A: B — 無具體場景，P1 群組事件只新增 enum 存根 + label，不實作觸發邏輯
- Q: LINE_BEACON 是否需要加入？ → A: C — 完全不加 LINE_BEACON，連 enum 存根都不加
- Q: postback_action 過濾值的匹配方式？ → A: B — 任意字串匹配，params 中填 postback_action 字串即可過濾，不綁定 EAction enum
- Q: LINE_POSTBACK_RECEIVED context 中 postback_data 的格式？ → A: A — postback_data 存 postback.data 的原始字串，postback_action 存解析後的 action 值
- Q: postback_action 為空或未填時的行為？ → A: A — 不過濾，所有 Postback 事件都觸發（寬鬆模式）
