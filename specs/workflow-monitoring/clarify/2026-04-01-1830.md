# Clarify Session 2026-04-01 18:30

## Idea

Workflow 清單頁新增 ID 欄與用戶欄。

Power Funnel 的 Workflow 監控清單頁（#/workflows）已有基本功能（Phase 1+2 已完成），但管理員反映缺少兩個關鍵欄位：

1. Workflow ID：方便引用特定 workflow（「ID 1100 那筆」）
2. 觸發用戶：判斷 workflow 由哪個用戶的行為觸發（已登入顯示 display_name，未登入或系統觸發顯示「訪客」）

後端需要在 Workflow 建立時記錄觸發用戶（post_author），清單 API 回傳新欄位，前端表格新增兩欄。

## Q&A

- Q: post_author 記錄的是終端用戶（如 Alice）還是操作者（如管理員 Bob）？ → A: 選 A，記錄終端用戶（Alice）。理由：後續 workflow 節點可能串接發信或發 LINE，對象是終端用戶。context resolve 方法也應在回傳的 context array 中包含 trigger_user_id 欄位。訪客（未登入）則 post_author = 0。
- Q: create_from() 如何取得 trigger_user_id？ → A: 選 B，新增 int $trigger_user_id = 0 參數，由 caller（TriggerPointService）傳入。不在 create_from() 內 resolve context，也不用 get_current_user_id()。create_failed_from_recursion_exceeded() 同步新增此參數。
