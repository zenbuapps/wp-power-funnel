# Clarify Session 2026-04-02 11:00

## Idea

觸發時機點 TriggerPoint 分組 + 搜尋功能。

目前 ETriggerPoint 已擴展至 20 個觸發點（含 5 個枚舉存根），前端 Select 組件以扁平列表方式呈現所有選項。隨著觸發點數量增長，使用者難以快速定位目標選項。

需求：
- 後端 API 回傳觸發點時包含分組資訊（group 欄位）
- 前端 Select 使用 OptGroup 依分類呈現觸發點選項
- Select 加入 showSearch 搜尋功能，支援模糊匹配觸發點名稱
- 所有使用觸發點 Select 的地方（Edit + List）同步更新
- REGISTRATION_CREATED 不出現在選項中

## Q&A

- Q: 5 個還沒實作的枚舉存根（LINE_JOIN、LINE_LEAVE、LINE_MEMBER_JOINED、LINE_MEMBER_LEFT、ACTIVITY_ENDED、PROMO_LINK_CLICKED），Select 裡要顯示嗎？ → A: 顯示但標 disabled +「（即將推出）」
- Q: 搜尋時輸入關鍵字，要同時比對「分組名稱」和「選項名稱」嗎？ → A: 兩個都比對
- Q: API 回傳格式：維持扁平陣列還是分組結構？ → A: 不需要向下兼容，直接用分組結構（nested grouped structure），例如 `{ group: string, group_label: string, items: {hook, name, disabled}[] }[]`
