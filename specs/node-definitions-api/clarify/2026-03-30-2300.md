# Clarify Session 2026-03-30 23:00

## Idea

目前工作流節點編輯器的節點選單（CustomEdge 的 Handle OnClick）與 Drawer 表單，全部寫死在前端。需要改為後端 API 驅動。

1. 新增 REST API `GET /wp-json/power-funnel/node-definitions` -- 回傳所有已註冊的 NodeDefinition（含 form fields schema）
2. 補齊 10 種 NodeDefinition -- 將 ENode enum 的 10 個 case 全部實例化為 BaseNodeDefinition 子類別並註冊
3. 每個 NodeDefinition 包含 form_fields schema -- 描述該節點 Drawer 表單的欄位定義（type、label、name、rules、options 等）
4. 前端改為動態渲染 -- 根據 API 回傳的 form_fields schema 動態生成 Drawer 表單，取代目前的 switch + 硬編碼 Form 元件
5. 第三方可擴充 -- 透過 power_funnel/workflow_rule/node_definitions filter hook 註冊自訂節點，前端自動支援

## Q&A

- Q: FormFieldDTO 結構：沿用 Powerhouse vs 新建 vs 純 array? → A: B -- 擴充 Powerhouse 的 FormFieldDTO，新增 name, label, type, required, default_value, placeholder, description, validation, sort, depends_on 等屬性。Powerhouse 的 FormFieldDTO 目前只被 power-funnel 使用，擴充不影響其他外掛。
- Q: 前端 DynamicNodeForm 取代策略：完全取代 vs 保留特殊 Form vs 漸進式? → A: A -- 完全取代，移除 EmailForm、WaitForm、DefaultForm，統一由 DynamicNodeForm 根據 form_fields schema 動態渲染。
- Q: 10 個節點各自的 form_fields 具體欄位定義? → A: 接受提案。email(recipient,subject_tpl,content_tpl,message_tpl_id), sms(recipient,content_tpl), line(content_tpl), webhook(url,method,headers,body_tpl), wait(duration,unit), wait_until(datetime), time_window(start_time,end_time,timezone), yes_no_branch(condition_field,operator,condition_value), split_branch(branches), tag_user(tags,action)。
