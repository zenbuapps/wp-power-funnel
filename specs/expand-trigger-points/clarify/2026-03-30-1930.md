# Clarify Session 2026-03-30 19:30

## Idea

擴展工作流觸發時機點。目前 ETriggerPoint enum 僅有 REGISTRATION_CREATED 一個觸發點，
需新增五類共 15 個觸發點：

- P0：報名狀態變更（REGISTRATION_APPROVED / REJECTED / CANCELLED / FAILED）
- P1：LINE 互動事件（LINE_FOLLOWED / UNFOLLOWED / MESSAGE_RECEIVED）
- P2：工作流引擎事件（WORKFLOW_COMPLETED / WORKFLOW_FAILED）+ 遞迴防護
- P3：活動時間事件（ACTIVITY_STARTED / ENDED / BEFORE_START）
- P3：用戶行為事件（USER_TAGGED / PROMO_LINK_CLICKED）

五類全部寫入 specs，標注優先級，實作時依序進行。

## Q&A

- Q: REGISTRATION_APPROVED 是否對應 post_status = success？ → A: 是，用戶確認。pending 為待審核，success 為審核通過。
- Q: 此次 specs 範圍？(A) 只做 P0 (B) P0+P1 (C) 五類全部 → A: 選擇 (C)，五類全部一次寫入 specs，標注優先級。
- Q: 遞迴防護策略？(A) max_depth=3 靜默跳過 (B) max_depth=5 記 log 跳過 (C) max_depth=3 建立 failed 實例+記 log → A: 選擇 (C)，max_depth=3，超過時建立 status=failed 的 Workflow 實例 + 記 error log。
- Q: 活動時間事件的排程策略？(A) Action Scheduler (B) WP Cron (C) 混合 → A: 選擇 (A)，使用 Action Scheduler as_schedule_single_action 排程，活動同步時建立排程，更新時取消舊的重新排。
- Q: ActivityDTO 缺少結束時間，ACTIVITY_ENDED 如何處理？(A) 新增 scheduled_end_time (B) start+duration (C) 僅定義 enum case 暫不實作 → A: 選擇 (C)，僅定義 ETriggerPoint enum case，暫不實作排程觸發邏輯。
- Q: ACTIVITY_BEFORE_START 提前時間設定方式？(A) 固定 30 分鐘 (B) WorkflowRule trigger_point meta 配置 before_minutes (C) 全域設定 → A: 選擇 (B)，在 WorkflowRule 的 trigger_point meta 中配置 before_minutes 參數，trigger_point meta 從字串擴展為物件 {hook, params}。

- Q: PROMO_LINK_CLICKED 觸發機制？(A) LIFF 回調 (B) redirect 端點 (C) 僅定義 enum case 暫不實作 → A: 選擇 (C)，僅定義 ETriggerPoint enum case，暫不實作觸發邏輯，等點擊追蹤機制建立後再補。
